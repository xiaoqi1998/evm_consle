安全机制概述

本工具高度重视用户资产安全，采用业界领先的加密技术和分片存储方案，确保您的私钥永远不会以明文形式暴露。

私钥分片
为什么输入私钥是安全的？

1. 私钥 never 离开您的浏览器

所有加密操作都在您的本地浏览器中完成，私钥 never 以明文形式发送到服务器。

2. Shamir's Secret Sharing 分片技术

私钥被分割成多个片段，分别存储在浏览器本地和服务器。即使攻击者同时获取两端数据，也无法还原原始私钥。

3. AES-256-GCM 强加密

浏览器端使用 AES-256-GCM 算法加密私钥片段，密钥由您的密码通过 PBKDF2 派生，强度极高。

4. 服务器无法还原私钥

服务器仅存储加密后的片段，且缺少浏览器端的加密密钥，因此无法还原原始私钥。

Shamir's Secret Sharing 分片算法

Shamir's Secret Sharing 是一种密码学算法，可以将一个秘密（如私钥）分割成多个片段，只有达到一定数量的片段才能还原原始秘密。

工作原理：
  • 将私钥分割成 N 个片段
  • 设置阈值 K（K ≤ N）
  • 需要至少 K 个片段才能还原私钥
  • 少于 K 个片段无法获取任何信息
分片存储策略：
浏览器端

存储内容：加密后的私钥片段

加密密钥：由您的密码派生（PBKDF2）

安全性：只有您知道密码，密钥无法被破解

服务器端

存储内容：未加密的私钥片段

密钥：无

安全性：单独的片段无法还原私钥

实际应用示例：
// 将私钥分割成 3 个片段，需要至少 2 个片段才能还原
const privateKey = "0x1234567890abcdef..."; // 256 位私钥
const shares = Shamir.share(privateKey, 3, 2);
// shares = ["1-abc...", "2-def...", "3-ghi..."]

// 存储策略：
// - 片段 1: 加密后存储在浏览器 localStorage
// - 片段 2: 以明文形式存储在服务器数据库
// - 片段 3: 可选存储在其他位置（如用户自己保管）

// 还原私钥（需要至少 2 个片段）
const combined = Shamir.combine(["1-abc...", "2-def..."]);
// combined = "0x1234567890abcdef..."
AES-256-GCM 加密算法

AES-256-GCM 是一种先进的对称加密算法，提供机密性、完整性和认证性保护。

256 位密钥

2^256 种可能，暴力破解需要宇宙年龄的时间

GCM 模式

提供认证加密，防止密文被篡改

随机 IV

每次加密使用不同的初始化向量

密钥派生（PBKDF2）：

您的登录密码通过 PBKDF2 算法派生出 AES 加密密钥，包含以下安全特性：

  • 600,000 次迭代：增加暴力破解成本
  • Salt 值：防止彩虹表攻击
  • SHA-256 哈希：确保密钥强度
// 密钥派生过程
async function deriveKey(password, salt) {
    const encoder = new TextEncoder();
    const baseKey = await window.crypto.subtle.importKey(
        "raw", 
        encoder.encode(password), 
        "PBKDF2", 
        false, 
        ["deriveKey"]
    );
    
    return window.crypto.subtle.deriveKey(
        { 
            name: "PBKDF2", 
            salt: encoder.encode(salt),  // 使用账户别名作为 salt
            iterations: 600000,          // 60 万次迭代
            hash: "SHA-256"              // SHA-256 哈希
        },
        baseKey,
        { name: "AES-GCM", length: 256 },  // 256 位 AES 密钥
        false,
        ["encrypt", "decrypt"]
    );
}

1. 私钥加密存储（saveEncryptedSlice）

// 加密分片并存入 localStorage
async saveEncryptedSlice(alias, sliceB, password) {
    try {
        const iv = window.crypto.getRandomValues(new Uint8Array(12));
        const key = await this.deriveKey(password, alias); // 别名作为盐值
        
        const encrypted = await window.crypto.subtle.encrypt(
            { name: "AES-GCM", iv },
            key,
            new TextEncoder().encode(sliceB)
        );
        
        // 将 IV 和密文拼接存储
        const result = {
            iv: Array.from(iv),
            data: Array.from(new Uint8Array(encrypted))
        };
        
        localStorage.setItem(`account_${alias}_sliceB_enc`, JSON.stringify(result));
        
        // 加密成功后自动缓存 Session Key（30分钟）
        this._setSessionKey(alias, key);
        
        return true;
    } catch (error) {
        console.error('加密分片失败:', error);
        throw new Error('加密分片失败：' + error.message);
    }
}
2. 私钥解密读取（getDecryptedSlice）
// 解密本地分片（核心逻辑修改：支持 Session 缓存）
async getDecryptedSlice(alias, password = null) {
    try {
        const stored = localStorage.getItem(`account_${alias}_sliceB_enc`);
        if (!stored) {
            throw new Error("未找到加密分片，请确认是否清除了浏览器缓存");
        }
        
        const parsed = JSON.parse(stored);

        // 1. 先尝试从 Session 缓存获取 Key（30分钟有效）
        let key = this._getSessionKey(alias);

        // 2. 如果没有缓存的 Key 且用户没传密码，抛出 SESSION_EXPIRED 错误
        if (!key) {
            if (!password) {
                throw new Error("SESSION_EXPIRED");
            }
            // 使用传入的密码派生 Key
            key = await this.deriveKey(password, alias);
            // 缓存 Key
            this._setSessionKey(alias, key);
        }

        const decrypted = await window.crypto.subtle.decrypt(
            { name: "AES-GCM", iv: new Uint8Array(parsed.iv) },
            key,
            new Uint8Array(parsed.data)
        );
        
        return new TextDecoder().decode(decrypted);
    } catch (error) {
        // 如果解密失败（可能是密码错误、Session 被劫持或数据损坏）
        if (error.name === 'CryptoError' || error.name === 'InvalidCharacterError') {
            this._clearSession();
            throw new Error('密码错误或分片损坏');
        }
        // 如果是其他错误（如 SESSION_EXPIRED），直接抛出
        throw error;
    }
}
3. Session 缓存机制
const SecureStorage = {
    // 内存缓存：不存密码，只存派生出的密钥对象和过期时间
    _session: {
        key: null,
        expiresAt: 0,
        alias: null
    },

    // 内部方法：获取有效的 Session Key
    _getSessionKey(alias) {
        const now = Date.now();
        if (this._session.key && this._session.alias === alias && now < this._session.expiresAt) {
            return this._session.key;
        }
        return null;
    },

    // 内部方法：设置 Session Key（30 分钟有效期）
    _setSessionKey(alias, key) {
        this._session = {
            key: key,
            alias: alias,
            expiresAt: Date.now() + (30 * 60 * 1000) // 30 分钟
        };
        console.log(`[SecureStorage] Session 已设置，有效期至：${new Date(this._session.expiresAt).toLocaleTimeString()}`);
    },

    // 内部方法：清除 Session
    _clearSession() {
        this._session = {
            key: null,
            expiresAt: 0,
            alias: null
        };
        console.log('[SecureStorage] Session 已清除');
    },

    // 手动清除 Session（用于用户主动锁定）
    clearSession() {
        this._clearSession();
    },

    // 检查 Session 是否有效
    isSessionValid(alias) {
        const now = Date.now();
        return this._session.key && this._session.alias === alias && now < this._session.expiresAt;
    }
};
关键安全特性：
  • 密码 never 存储：密码仅用于派生密钥，不保存在任何地方
  • Session 自动过期：30 分钟后需要重新输入密码
  • 手动锁定功能：用户可以主动清除 Session
  • 错误处理机制：密码错误会清除 Session，防止暴力破解

常见问题
Q: 我的私钥会发送到服务器吗？

A: 不会。您的私钥永远不会发送到服务器，所有加密操作都在浏览器端完成。

Q: 服务器可以还原我的私钥吗？

A: 不可以。服务器只存储加密后的分片，且缺少浏览器端的加密密钥，无法还原原始私钥。

Q: 我的密码会存储在服务器吗？

A: 不会。密码仅用于在浏览器端派生加密密钥，不会以任何形式存储或传输到服务器。

Q: 如果我忘记密码怎么办？

A: 非常抱歉，由于我们采用端到端加密，密码无法被找回。建议您妥善保管密码，并考虑使用密码管理器。

Q: 可以证明代码没有后门吗？

A: 本工具完全开源，所有代码均可在 GitHub 上查看。您可以自行审查代码，或使用编译后的版本。我们承诺永远不会在代码中添加任何后门或隐藏功能。
