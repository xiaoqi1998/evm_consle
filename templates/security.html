安全机制概述

本工具采用多重安全防护机制，通过端到端加密、私钥分片和强加密算法，确保您的私钥和资产安全。所有敏感操作均在浏览器本地完成，私钥永不离开您的设备。

为什么输入私钥是安全的？

1. 端到端加密保护

所有加密操作均在浏览器本地完成，私钥永不离开您的设备，确保端到端的安全性。

2. 私钥分片存储

采用 Shamir's Secret Sharing 技术将私钥分割存储，即使部分数据泄露也无法还原完整私钥。

3. 强加密算法

使用 AES-256-GCM 算法加密，配合 PBKDF2 密钥派生，提供军事级别的加密保护。

4. 零信任架构

服务器仅存储无法单独使用的分片，采用零信任原则，确保任何单点都无法获取完整私钥。

Shamir's Secret Sharing 分片算法

Shamir's Secret Sharing 是一种先进的密码学算法，通过多项式插值技术将私钥安全地分割成多个片段，只有收集到足够数量的片段才能还原原始私钥。

工作原理：
  • 将私钥分割成 N 个片段
  • 设置阈值 K（K ≤ N）
  • 需要至少 K 个片段才能还原私钥
  • 少于 K 个片段无法获取任何信息
分片存储策略：
浏览器端

存储内容：加密后的私钥片段

加密密钥：由您的密码派生（PBKDF2）

安全性：只有您知道密码，密钥无法被破解

服务器端

存储内容：未加密的私钥片段

密钥：无

安全性：单独的片段无法还原私钥

实际应用示例：
// 将私钥分割成 3 个片段，需要至少 2 个片段才能还原
const privateKey = "0x1234567890abcdef..."; // 256 位私钥
const shares = Shamir.share(privateKey, 3, 2);
// shares = ["1-abc...", "2-def...", "3-ghi..."]

// 存储策略：
// - 片段 1: 加密后存储在浏览器 localStorage
// - 片段 2: 以明文形式存储在服务器数据库
// - 片段 3: 可选存储在其他位置（如用户自己保管）

// 还原私钥（需要至少 2 个片段）
const combined = Shamir.combine(["1-abc...", "2-def..."]);
// combined = "0x1234567890abcdef..."
二维码密保体系 (QR Code Backup)

二维码密保体系是本工具的创新安全机制，通过二维码技术实现私钥分片的便捷备份和恢复。

工作原理：
  • 将切片 C（密码分片）封装成 JSON 格式并生成二维码
  • 二维码包含账户别名、版本号和切片内容
  • 支持离线保存，完全在前端生成，不经过服务器
  • 恢复时通过扫码或上传图片识别二维码
技术方案：
备份阶段
  • 将切片 C 封装成 JSON：{"v":1, "a":"别名", "s":"切片内容"}
  • 使用 qrcode.js 生成二维码
  • 显示账户地址和"密保凭证"水印
  • 支持下载为 PNG 图片
恢复阶段
  • PC端：使用 jsQR 上传图片识别
  • 手机端：使用 html5-qrcode 调用摄像头
  • 解析 JSON 获取切片 C
  • 向服务器请求切片 A
  • 本地重构：A + C = 私钥
安全增强：
重要安全提示：
  • 二维码生成逻辑完全在前端完成，绝不能将切片 C 发送到后端让后端生成二维码图片
  • 建议打印二维码或存入物理加密U盘，不要存入手机相册或云端
  • 一旦二维码泄露，黑客在获取登录权限后可直接盗取资金
  • 恢复时会重新生成新的分片，旧的二维码将失效
使用流程：
  1. 添加账户时，系统自动生成密保二维码
  2. 点击"下载密保图片"保存为 PNG 文件
  3. 半年后设备重装系统，本地分片丢失
  4. 登录账号，点击"恢复"按钮，上传二维码图片
  5. 系统自动同步切片 A，重新生成分片并设置新密码
AES-256-GCM 加密算法

AES-256-GCM 是业界公认的强加密算法，提供机密性、数据完整性和认证保护，被广泛应用于军事和金融领域。

256 位密钥

2^256 种可能，暴力破解需要宇宙年龄的时间

GCM 模式

提供认证加密，防止密文被篡改

随机 IV

每次加密使用不同的初始化向量

密钥派生（PBKDF2）：

您的登录密码通过 PBKDF2 密钥派生函数生成强加密密钥，具备以下安全特性：

  • 600,000 次迭代：增加暴力破解成本
  • Salt 值：防止彩虹表攻击
  • SHA-256 哈希：确保密钥强度
// 密钥派生过程
async function deriveKey(password, salt) {
    const encoder = new TextEncoder();
    const baseKey = await window.crypto.subtle.importKey(
        "raw", 
        encoder.encode(password), 
        "PBKDF2", 
        false, 
        ["deriveKey"]
    );
    
    return window.crypto.subtle.deriveKey(
        { 
            name: "PBKDF2", 
            salt: salt,  // 使用加密学安全的随机 salt
            iterations: 600000,          // 60 万次迭代
            hash: "SHA-256"              // SHA-256 哈希
        },
        baseKey,
        { name: "AES-GCM", length: 256 },  // 256 位 AES 密钥
        false,
        ["encrypt", "decrypt"]
    );
}
SecureStorage 模块源码

以下是从 crypto-utils.js 中提取的 SecureStorage 核心代码，展示了私钥的加密存储过程：

1. 私钥加密存储（saveEncryptedSlice）
// 使用 PBKDF2 算法从密码派生出强密钥
async deriveKey(password, salt) {
    const encoder = new TextEncoder();
    const baseKey = await window.crypto.subtle.importKey(
        "raw", 
        encoder.encode(password), 
        "PBKDF2", 
        false, 
        ["deriveKey"]
    );
    
    return window.crypto.subtle.deriveKey(
        { 
            name: "PBKDF2", 
            salt: encoder.encode(salt), 
            iterations: 600000, 
            hash: "SHA-256" 
        },
        baseKey,
        { name: "AES-GCM", length: 256 },
        false,
        ["encrypt", "decrypt"]
    );
}

// 加密分片并存入 localStorage
async saveEncryptedSlice(alias, sliceB, password) {
    try {
        const iv = window.crypto.getRandomValues(new Uint8Array(12));
        const salt = window.crypto.getRandomValues(new Uint8Array(16));  // 加密学安全的随机 salt
        const key = await this.deriveKey(password, salt);
        
        const encrypted = await window.crypto.subtle.encrypt(
            { name: "AES-GCM", iv },
            key,
            new TextEncoder().encode(sliceB)
        );
        
        // 将 Salt、IV 和密文拼接存储
        const result = {
            salt: Array.from(salt),  // 新增：存储 salt 用于解密
            iv: Array.from(iv),
            data: Array.from(new Uint8Array(encrypted))
        };
        
        localStorage.setItem(`account_${alias}_sliceB_enc`, JSON.stringify(result));
        
        // 加密成功后自动缓存 Session Key（30分钟）
        this._setSessionKey(alias, key);
        
        return true;
    } catch (error) {
        console.error('加密分片失败:', error);
        throw new Error('加密分片失败：' + error.message);
    }
}
2. 私钥解密读取（getDecryptedSlice）
// 解密本地分片（核心逻辑修改：支持 Session 缓存）
async getDecryptedSlice(alias, password = null) {
    try {
        const stored = localStorage.getItem(`account_${alias}_sliceB_enc`);
        if (!stored) {
            throw new Error("未找到加密分片，请确认是否清除了浏览器缓存");
        }
        
        const parsed = JSON.parse(stored);

        // 1. 先尝试从 Session 缓存获取 Key（30分钟有效）
        let key = this._getSessionKey(alias);

        // 2. 如果没有缓存的 Key 且用户没传密码，抛出 SESSION_EXPIRED 错误
        if (!key) {
            if (!password) {
                throw new Error("SESSION_EXPIRED");
            }
            // 使用传入的密码和存储的 salt 派生 Key
            const salt = new Uint8Array(parsed.salt);
            key = await this.deriveKey(password, salt);
            // 缓存 Key
            this._setSessionKey(alias, key);
        }

        const decrypted = await window.crypto.subtle.decrypt(
            { name: "AES-GCM", iv: new Uint8Array(parsed.iv) },
            key,
            new Uint8Array(parsed.data)
        );
        
        return new TextDecoder().decode(decrypted);
    } catch (error) {
        // 如果解密失败（可能是密码错误、Session 被劫持或数据损坏）
        if (error.name === 'CryptoError' || error.name === 'InvalidCharacterError') {
            this._clearSession();
            throw new Error('密码错误或分片损坏');
        }
        // 如果是其他错误（如 SESSION_EXPIRED），直接抛出
        throw error;
    }
}
3. Session 缓存机制
const SecureStorage = {
    // 内存缓存：不存密码，只存派生出的密钥对象和过期时间
    _session: {
        key: null,
        expiresAt: 0,
        alias: null
    },

    // 内部方法：获取有效的 Session Key
    _getSessionKey(alias) {
        const now = Date.now();
        if (this._session.key && this._session.alias === alias && now < this._session.expiresAt) {
            return this._session.key;
        }
        return null;
    },

    // 内部方法：设置 Session Key（30 分钟有效期）
    _setSessionKey(alias, key) {
        this._session = {
            key: key,
            alias: alias,
            expiresAt: Date.now() + (30 * 60 * 1000) // 30 分钟
        };
        console.log(`[SecureStorage] Session 已设置，有效期至：${new Date(this._session.expiresAt).toLocaleTimeString()}`);
    },

    // 内部方法：清除 Session
    _clearSession() {
        this._session = {
            key: null,
            expiresAt: 0,
            alias: null
        };
        console.log('[SecureStorage] Session 已清除');
    },

    // 手动清除 Session（用于用户主动锁定）
    clearSession() {
        this._clearSession();
    },

    // 检查 Session 是否有效
    isSessionValid(alias) {
        const now = Date.now();
        return this._session.key && this._session.alias === alias && now < this._session.expiresAt;
    }
};
关键安全特性：
  • 密码 never 存储：密码仅用于派生密钥，不保存在任何地方
  • Session 自动过期：30 分钟后需要重新输入密码
  • 手动锁定功能：用户可以主动清除 Session
  • 错误处理机制：密码错误会清除 Session，防止暴力破解

常见问题
Q: 我的私钥会发送到服务器吗？

A: 不会。我们采用端到端加密架构，私钥仅在您的浏览器本地处理，永远不会以任何形式发送到服务器。

Q: 服务器可以还原我的私钥吗？

A: 绝对不可以。服务器仅存储无法单独使用的私钥分片，且缺少浏览器端的加密密钥，技术上无法还原完整私钥。

Q: 我的密码会存储在服务器吗？

A: 不会。密码仅在浏览器本地用于派生加密密钥，不会存储或传输到任何服务器，确保密码的绝对安全。

Q: 如果我忘记密码怎么办？

A: 由于采用端到端加密设计，密码无法被找回或重置。我们强烈建议您使用密码管理器妥善保管密码，并定期备份重要数据。

Q: 可以证明代码没有后门吗？

A: 本工具采用完全开源模式，所有源代码均在 GitHub 公开。您可以自行审查代码，或使用社区验证的版本。我们郑重承诺永远不会在代码中添加任何后门、恶意功能或隐藏代码。
