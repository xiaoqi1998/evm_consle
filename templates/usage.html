工具概述

EVM API 控制面板是一个专业的 Web3 开发者工具，支持以太坊虚拟机(EVM)合约交互、ABI 管理、MetaMask 连接、交易解码及事件日志查询。支持多链节点切换，助力高效智能合约开发。

⚠️ 测试链建议： 本工具建议仅在测试链（Testnet）上使用，不要在主网（Mainnet）上处理大额资产。测试链上的操作不会影响真实资产，更适合开发和测试阶段。
ℹ️ 免费 RPC 说明： 默认配置的 RPC 地址多为免费公共节点，网络稳定性可能无法保证。如需更高稳定性，建议使用付费 RPC 服务或自建节点。
核心功能：
  • 智能合约交互（Read/Write 方法）
  • 多链 RPC 节点配置与切换
  • ABI 文件上传与管理
  • MetaMask 钱包连接
  • 交易数据解码
  • 合约事件日志查询
  • 调用历史记录管理
快速开始
使用步骤：
步骤 1：登录/注册

首先需要登录或注册一个账号，获取访问权限。

步骤 2：配置账户

在左侧边栏的"账户配置"部分添加您的钱包账户，包括别名、私钥和地址。

步骤 3：配置 RPC 节点

在"链/RPC 节点"部分添加您需要使用的区块链网络 RPC 节点。

步骤 4：上传 ABI

在"ABI 管理"部分上传您的智能合约 ABI 文件，以便后续调用合约方法。

步骤 5：开始合约交互

在"合约交互"标签页中，输入合约地址、选择 ABI 和链/RPC，然后加载并执行合约方法。

账户配置

账户配置用于存储您的钱包信息，方便在调用合约时选择使用。

添加账户：
  1. 点击左侧边栏"账户配置"下的"添加配置"按钮
  2. 在弹出的模态框中填写：
    • 别名：账户的昵称，方便识别
    • 私钥：钱包的私钥（用于签名交易）
    • 地址：钱包的地址
  3. 点击"保存账户"按钮
🔒 安全机制说明

您的私钥采用 Shamir's Secret Sharing 分片加密技术进行保护：

  • 私钥分片：私钥被分割成多个片段，分别存储在浏览器本地和服务器
  • 本地加密：浏览器端使用 AES-256-GCM 加密私钥片段，密钥由您的密码派生
  • 服务器存储：服务器仅存储加密后的私钥片段，无法还原原始私钥
  • 无需传输明文：私钥 never 以明文形式传输或存储在服务器

因此，即使服务器被攻击，攻击者也无法获取您的私钥！

⚠️ 安全提示：请妥善保管您的登录密码，密码用于派生加密密钥。如果忘记密码，将无法解密私钥。
🛡️ 开源透明：本工具完全开源，所有加密代码均可在 安全与开源页面 查看，确保无后门、无隐藏逻辑。
RPC 节点配置

RPC 节点配置用于连接不同的区块链网络，支持自定义 RPC 地址。

添加 RPC 节点：
  1. 点击左侧边栏"链/RPC 节点"下的"添加配置"按钮
  2. 在弹出的模态框中填写：
    • 链 ID：区块链网络的链 ID（如以太坊主网为 1）
    • 别名：网络的昵称，方便识别
    • RPC URL：RPC 节点的 URL 地址
  3. 点击"保存 RPC"按钮
支持的网络：
Ethereum BSC Polygon Arbitrum Optimism Base Avalanche Linea ... 任意 EVM 兼容链
ABI 管理

ABI（应用程序二进制接口）是与智能合约交互的关键，包含了合约的方法、事件等信息。

上传 ABI：
  1. 点击左侧边栏"ABI 管理"下的"上传 ABI"按钮
  2. 在弹出的模态框中填写：
    • ABI 名称：ABI 文件的名称，方便识别
    • ABI 内容：智能合约的 ABI JSON 数组
  3. 点击"上传"按钮
ABI 格式示例：
[
  {
    "inputs": [],
    "name": "totalSupply",
    "outputs": [
      {
        "internalType": "uint256",
        "name": "",
        "type": "uint256"
      }
    ],
    "stateMutability": "view",
    "type": "function"
  },
  {
    "inputs": [
      {
        "internalType": "address",
        "name": "account",
        "type": "address"
      }
    ],
    "name": "balanceOf",
    "outputs": [
      {
        "internalType": "uint256",
        "name": "",
        "type": "uint256"
      }
    ],
    "stateMutability": "view",
    "type": "function"
  }
]
合约交互

合约交互是本工具的核心功能，支持调用智能合约的读写方法。

使用步骤：
  1. 在"合约地址"输入框中填写智能合约的地址
  2. 从"ABI"下拉菜单中选择已上传的 ABI 文件
  3. 从"链/RPC"下拉菜单中选择要连接的网络
  4. 点击"加载方法"按钮，系统会解析并显示合约的所有方法
  5. 从左侧方法列表中选择要调用的方法
  6. 根据方法要求填写参数（如果有）
  7. 对于写入方法（Write），选择发送账户和设置 value（如果需要）
  8. 点击"执行方法"按钮
  9. 查看执行结果
方法类型：
  • Read 方法：无需签名，直接查询合约状态
  • Write 方法：需要签名交易，会改变合约状态
辅助工具

辅助工具提供了交易回执查询和解码功能，方便开发者调试。

获取交易回执：
  1. 在"获取交易回执"卡片中输入交易哈希
  2. 选择对应的链/RPC
  3. 点击"查询回执"按钮
  4. 查看交易的详细信息，包括状态、gas 使用情况等
解码交易数据：
  1. 在"解码交易数据"卡片中输入编码数据（Input Data）
  2. 选择对应的 ABI 文件
  3. 点击"解码数据"按钮
  4. 查看解码后的方法名和参数
事件日志查询

事件日志查询功能允许您查询智能合约发出的事件，方便跟踪合约活动。

使用步骤：
  1. 输入合约地址
  2. 选择对应的 ABI 文件
  3. 选择要连接的链/RPC
  4. 输入事件名称（如 Transfer）
  5. 设置起始区块和结束区块
  6. （可选）输入参数过滤条件（JSON 格式）
  7. 点击"查询事件"按钮
  8. 查看事件日志列表
参数过滤示例：
{"from": "0x1234567890123456789012345678901234567890"}
MetaMask 连接

本工具支持连接 MetaMask 钱包，方便直接使用钱包进行交易签名。

连接步骤：
  1. 点击左侧边栏"Web3 Wallet"下的"连接 MetaMask"按钮
  2. 在 MetaMask 弹出的窗口中授权连接
  3. 连接成功后，会显示当前连接的账户地址
提示：连接 MetaMask 后，您可以直接使用钱包中的账户进行交易签名，无需在工具中存储私钥。
调用历史记录

调用历史记录功能会保存您的合约调用历史，方便查看和重新调用。

功能说明：
  • 自动记录所有合约调用（包括成功和失败）
  • 显示调用时间、类型、方法、参数和结果
  • 支持重新调用历史记录中的方法
  • 支持刷新和清空历史记录
常见问题
Q&A：
Q: 连接 RPC 节点失败怎么办？

A: 请检查 RPC URL 是否正确，网络是否通畅，或者尝试使用其他 RPC 节点。

Q: 执行合约方法时出现错误？

A: 请检查：

  • 合约地址是否正确
  • ABI 文件是否与合约匹配
  • 参数是否正确
  • 账户余额是否足够支付 gas 费用
  • 网络是否拥堵

Q: 如何获取智能合约的 ABI？

A: 您可以从以下途径获取 ABI：

  • 从智能合约编译工具（如 Remix、Hardhat、Truffle）导出
  • 从 Etherscan 等区块浏览器获取已验证合约的 ABI
  • 从合约开发者处获取

Q: 工具支持哪些网络？

A: 本工具支持所有 EVM 兼容的区块链网络，包括但不限于 Ethereum、BSC、Polygon、Arbitrum、Optimism、Base、Avalanche、Linea 等。
