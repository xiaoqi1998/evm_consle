API 概述

EVM API 控制面板提供了一系列 RESTful API，用于管理账户、RPC 节点、ABI 文件，以及与智能合约进行交互。所有 API 都需要通过 X-API-Token 进行认证。

API 特性：
  • RESTful 设计风格
  • 统一的错误处理和响应格式
  • 支持多链节点切换
  • 完整的智能合约交互功能
  • 支持 MetaMask 集成
认证方式

所有 API 接口都需要通过 X-API-Token 进行认证。登录后，系统会返回一个 Token，您需要在每次请求时将该 Token 放在请求头中。

认证请求头：
X-API-Token: evm-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
使用示例：
// JavaScript 示例
fetch('/api/accounts', {
  headers: {
    'X-API-Token': 'evm-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'
  }
});
账户管理 API
GET /api/accounts
GET

获取当前用户的账户列表

响应格式：
{"status": "success", "message": "Success", "data": {"account1": {"address": "0x...", "private_key": "0x..."}, ...}}
POST /api/accounts
POST

添加新的账户

请求体：
{"alias": "account1", "address": "0x...", "private_key": "0x..."}
响应格式：
{"status": "success", "message": "账户已保存"}
DELETE /api/accounts/{alias}
DELETE

删除指定的账户

路径参数：
alias: 账户别名
响应格式：
{"status": "success", "message": "账户已删除"}
RPC 管理 API
GET /api/rpcs
GET

获取当前用户的 RPC 节点列表

响应格式：
{"status": "success", "message": "Success", "data": [{"chain_id": "1", "alias": "Ethereum Mainnet", "rpc_url": "https://..."}, ...]}
POST /api/rpcs
POST

添加新的 RPC 节点

请求体：
{"chain_id": 1, "alias": "Ethereum Mainnet", "rpc_url": "https://..."}
响应格式：
{"status": "success", "message": "RPC 已保存"}
DELETE /api/rpcs/{chain_id}
DELETE

删除指定的 RPC 节点

路径参数：
chain_id: 链 ID
响应格式：
{"status": "success", "message": "RPC 已删除"}
ABI 管理 API
POST /upload_abi
POST

上传新的 ABI 文件

请求体：
{"abi_name": "Token", "abi_content": [...]}
响应格式：
{"status": "success", "message": "ABI 'Token' uploaded successfully"}
DELETE /delete_abi
DELETE

删除指定的 ABI 文件

查询参数：
abi_name: ABI 名称
响应格式：
{"status": "success", "message": "ABI 'Token' deleted successfully"}
GET /list_abis
GET

列出所有可用的 ABI 文件

响应格式：
{"status": "success", "message": "Success", "data": {"abis": ["Token", "ERC20", ...]}}
GET /get_abi_details
GET

获取 ABI 的详细信息，包括函数、事件、错误等分类

查询参数：
abi_name: ABI 名称
响应格式：
{"status": "success", "message": "Success", "data": {"details": {...}, "full_abi": [...]}}
合约交互 API
POST /read_contract
POST

调用合约的只读方法（View/Pure）

请求体：
{"contract_address": "0x...", "abi_name": "Token", "method_name": "balanceOf", "args": ["0x..."], "chain_id": 1}
响应格式：
{"status": "success", "message": "Success", "data": {"result": "1000000000000000000"}}
POST /write_contract
POST

调用合约的写入方法（改变合约状态）

请求体：
{"contract_address": "0x...", "abi_name": "Token", "method_name": "transfer", "args": ["0x...", "1000000000000000000"], "chain_id": 1, "account_alias": "account1"}
响应格式：
{"status": "success", "message": "Success", "data": {"tx_hash": "0x..."}}
事件查询 API
POST /get_events
POST

查询指定合约的事件日志

请求体：
{"contract_address": "0x...", "abi_name": "Token", "event_name": "Transfer", "from_block": 0, "to_block": "latest", "argument_filters": {"from": "0x..."}, "chain_id": 1}
响应格式：
{"status": "success", "message": "Success", "data": {"event_logs": [...]}}

POST

历史记录 API
GET /api/history
GET

获取当前用户的调用历史记录

响应格式：
{"status": "success", "message": "Success", "data": [{"id": "...", "timestamp": "...", "type": "read", "contract": "0x...", "method": "balanceOf", "args": [...], "result": "..."}, ...]}
POST /api/history/clear
POST

清空当前用户的调用历史记录

响应格式：
{"status": "success", "message": "History cleared"}

GET
